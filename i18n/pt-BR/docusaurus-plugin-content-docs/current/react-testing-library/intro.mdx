A [`React Testing Library`][gh] é criada em cima do `DOM Testing Library`
adicionando APIs para trabalhar com componentes React.

> Projetos criados com [`Create React App`](https://create-react-app.dev/)
> possuem suporte para a **React Testing Library** out of the box. Se não é o
> seu caso, você pode adicioná-la via `npm` assim:

```
npm install --save-dev @testing-library/react
```

- [React Testing Library no GitHub][gh]

[gh]: https://github.com/testing-library/react-testing-library

## O problema

Você quer escrever testes sustentáveis para seus componentes React. Como parte
desse objetivo, você quer evitar que seus testes incluam detalhes da
implementação do seu componente e focar em fazer seus testes te darem confiança
para o que eles são direcionados. Como parte disso, você quer que sua base de
testes seja sustentável no longo prazo para que refactors nos seus componentes
(mudanças na implementação, não nas funcionalidades) não quebrem seus testes e
deixem você e seu time mais lentos.

## Esta solução

A `React Testing Library` é uma solução muito leve para testar componentes
React. Ela te dá funções de utilidades leve em cima do `react-dom` e
`react-dom/test-utils`, de uma forma que encoraja melhores práticas de teste.
Seu princípio principal é:

> [Quanto mais seus testes parecem com o seu software sendo utilizado, mais confiança eles podem te dar.](guiding-principles.mdx)

Então ao invés de lidar com instâncias de componentes React renderizados, seus
testes vão, na verdade, funcionar com nós reais do DOM. As utilidades desta
biblioteca tornam as consultas no DOM iguais as que o usuário faria. Encontrar
elementos de um form pelos textos das labels (assim como o usuário faria),
encontrar links e botões a partir dos seus textos (assim como o usuário faria).
Ela também te dá uma forma recomendada de encontrar elementos por um
`data-testid` como uma "saída de emergência" para elementos onde o texto do
conteúdo ou sua label não fazem sentido ou não são práticos.

Essa biblioteca encoraja sua aplicação a ser mais acessível e te permite ter
seus testes mais próximos ao usar componentes de uma forma que o usuário vai
normalmente utiliza, o que permite com que seus testes te deem mais confiança de
que sua aplicação vai funcionar quando um usuário de verdade usá-la.

Essa biblioteca é uma substituição para o [Enzyme](http://airbnb.io/enzyme/).
Você _pode_ seguir esses guidelines usando o próprio Enzyme, mas saiba que isso
é mais difícil por causa de todas as utilidades extras que o Enzyme te dá
(utilidades as quais facilitam testes dos detalhes da implementação). Leia mais
sobre isso em nosso [FAQ](react-testing-library/faq.mdx).

**O que essa biblioteca não é**:

1.  Um test runner ou framework
2.  Específica para um framework de testes (nós recomendamos Jest como a nossa
    preferência, a biblioteca funciona com qualquer framework. Veja
    [Usando sem o Jest](setup.mdx#using-without-jest))

> NOTA: essa biblioteca é construída em cima do
> [`DOM Testing Library`](dom-testing-library/intro.mdx) que é onde a maior
> parte da lógica por trás das queries está.
